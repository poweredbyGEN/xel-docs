The current testnet package uses the recoverable shared mint entrypoints documented in
`docs/04-smart-contracts.md`: `mint_direct_prompt_shared` and `mint_classified_prompt_shared`.
The local package passes `sui move test --path contracts/v1 --build-env testnet` with shared mint
plus `finalize_recovery_and_transfer`.

This package adds the contract-first Endowment Waterfall:

- generic `Endowment<T>` with contract-held principal and ring-fenced keep-alive, storage, and
  interaction allocations,
- public top-up to principal or a named allocation,
- owner-only principal withdrawal above the keep-alive reserve floor,
- authority-free heartbeat that allocates realized yield by priority, compounds residual, and pays
  a fixed keeper bounty,
- owner-configured yield harvest policy and main treasury wallet ref,
- owner-configured yield venue allowlist by hash,
- owner-configured waterfall rates,
- monthly yield harvest intent with cadence/min-claim/destination/venue checks,
- epoch waterfall accounting event,
- creator-paid interaction split event.

Backend/keepers can execute PTBs and index events, but these policy rules now live on-chain.
